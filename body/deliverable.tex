\section{Software Solution}
\label{section:software-solution}
Following an iterative development process, we have successfully deployed an integrated system comprising a mobile application, a detection model, and a supporting infrastructure. The core of the system is a parking occupancy model that analyzes video feeds to evaluate and broadcast the spot vacancy status. This system leverages Computer Vision and Convolutional Neural Networks (CNNs) as detailed in Chapter 4 and Chapter 5 to transform raw visual data into actionable navigation insights for the end user.

Our repositories are available at https://github.com/ku-parking. The following sections of this chapter provide an overview of our mobile application.
\subsection{Map Overview}
\label{subsection:map-overview}
The Map Overview serves as the initial screen when a user enters the application. As shown in Figure \ref{fig:map-overview-screenshot}, the interface follows a typical map-navigation layout with a map serving as the primary source of information. The map displays the user's current location alongside parking area markers that indicate availability through both numerical values and color-coding. The color indicators are defined by the percentage of vacant spots relative to total capacity: Green represents vacancy of 70\% or higher, Yellow represents vacancy more than or equal 30\% but less than 70\%, and Red represents vacancy below 30\%. At the top of the screen, a search bar allows users to locate specific parking areas. At the bottom, a sheet drawer provides a list of all available areas, which users can select to view more granular details.

\subsection{Parking Spot Detail}
\label{subsection:parking-spot-detail}
When a user selects a specific parking area via the bottom sheet or the search bar, the detailed view for that location is displayed. As shown in Figure \ref{fig:parking-spot-detail-screenshot}, these details include the parking area name, the current number of vacant spots, and a timestamp indicating when the information was last updated. The \"Navigation\" button redirects users to their device’s native navigation application (such as Apple Maps or Google Maps) to provide directions to the selected parking area. Additionally, the \"Report Inaccurate Info\" button allows users to flag discrepancies in the data, a process which is discussed in greater detail in the following section.
